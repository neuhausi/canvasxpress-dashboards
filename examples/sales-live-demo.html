Phase 2 — authenticated connector binding (demo)

This page injects a fake connector backend via options.fetch so you can see the full data-binding flow with no real database. In production the same spec talks to canvasxpress-connectors over a cookie-authenticated /api/data endpoint — the only difference is you omit options.fetch and let the browser use the real one.

Watch for: a brief Loading… overlay per panel · the Live sales bar & pie share one request and auto-refresh every 3s (values change) · an empty source shows “No data” · a failing source shows the connector’s error detail.
